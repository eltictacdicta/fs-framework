# Quick Task: Corregir vendor incompleto Twig/Yaml tras instalación

## Diagnóstico

Fatal error tras instalación:

```
Interface "Twig\Node\CoercesChildrenToStringInterface" not found
```

## Causa raíz

El repositorio versiona `vendor/` para producción (~5100 archivos), pero al actualizar `twig/twig` v3.26.0 y `symfony/yaml` v7.4.12 no se commitearon 2 archivos nuevos:

- `vendor/twig/twig/src/Node/CoercesChildrenToStringInterface.php`
- `vendor/symfony/yaml/ParserState.php`

`composer install` responde "Nothing to install" porque los paquetes ya están presentes, pero no restaura archivos faltantes dentro de un paquete parcialmente versionado.

## Tareas

### Task 1: Reparar vendor y añadir archivos faltantes a git

**Action:** Reinstalar paquetes, añadir los 2 archivos al índice git.

**Verify:** Ambos archivos existen y `interface_exists('Twig\Node\CoercesChildrenToStringInterface')` es true.

### Task 2: Script de verificación post-instalación

**Action:** Crear `scripts/verify-vendor-integrity.sh` e integrarlo en README y `build.sh`.

**Verify:** Simular archivo faltante → script lo repara automáticamente.
