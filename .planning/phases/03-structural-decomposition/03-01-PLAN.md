# Plan 03-01: Delegate SHA1 Password Verification to legacy_support

## Objective

Remove SHA1/MD5 password verification fallback from `PasswordHasherService` core. The core should only handle modern hashes (bcrypt/argon2). All legacy hash verification (SHA1, MD5) must live exclusively in the `legacy_support` plugin, which is already the designated compatibility layer.

## must_haves

- truths:
  - `legacy_support` plugin already handles SHA1 migration via `LegacyCompatibility::verifyAndUpgradeLegacyPassword()`
  - `fs_login.php:408-422` already checks `class_exists('FSFramework\\Plugins\\legacy_support\\LegacyCompatibility')` before falling back
  - `PasswordHasherService::verifyLegacyHash()` (lines 232-251) is the private method that contains direct SHA1/MD5 comparison logic
  - `model/core/fs_user.php:525-530` has `is_legacy_sha1_password()` — must remain as detection method
  - The `isLowercasedLegacySha1Bypass()` method at lines 253-276 is a security gate that blocks certain legacy hash patterns — must be preserved in core or moved to legacy_support
- risks:
  - Removing the fallback without `legacy_support` plugin active would block login for legacy users
  - `fs_user::is_legacy_sha1_password()` is called from `fs_login.php:414` — removing it breaks the detection chain
  - `isLowercasedLegacySha1Bypass()` is a security hardening method that blocks known-weak patterns — it should stay in core even if verification moves

## Tasks

### Task 1: Move SHA1/MD5 verification from PasswordHasherService to LegacyCompatibility

<read_first>
- src/Security/PasswordHasherService.php (lines 230-276)
- plugins/legacy_support/LegacyCompatibility.php
</read_first>

<action>
In `src/Security/PasswordHasherService.php`:

1. Replace the private `verifyLegacyHash()` method (lines 232-251) with a version that ONLY delegates to `legacy_support`:

```php
private function verifyLegacyHash(
    string $storedHash,
    string $plainPassword,
    ?string $legacySalt = null
): bool {
    if (class_exists('FSFramework\\Plugins\\legacy_support\\LegacyCompatibility')) {
        return \FSFramework\Plugins\legacy_support\LegacyCompatibility::verifyLegacyPassword(
            $storedHash,
            $plainPassword,
            $legacySalt
        );
    }

    return false;
}
```

2. Keep `isLowercasedLegacySha1Bypass()` in PasswordHasherService — it's a security gate, not a legacy verifier.

3. Keep `isModernHash()` and `verifyWithLegacySupport()` methods — they're the public API that routes to legacy_support.

In `plugins/legacy_support/LegacyCompatibility.php`:

4. Add the SHA1/MD5 comparison logic (the old fallback code) to the `verifyLegacyPassword()` method if not already present. The method must handle:
   - `sha1($legacySalt . $plainPassword)` for salted SHA1
   - `sha1($plainPassword)` for unsalted SHA1
   - `md5($plainPassword)` for MD5
</action>
