# Plan 01-01: Update PHP Version Guards to 8.2

## Objective

Update the PHP version checks in `index.php` and `install.php` from 5.6 to 8.2. The current checks are dead code (Composer enforces the real constraint) but the displayed messages are incorrect and misleading.

## must_haves

- truths:
  - Composer already enforces PHP >=8.2 at install time
  - These guards are a user-facing fallback only (for when Composer hasn't run yet)
  - The version check logic must use `version_compare()` not float casting
- risks:
  - Using float casting (`floatval(substr(phpversion(), 0, 3))`) breaks for PHP 10+ (returns `1.` instead of `10`)
  - The install.php check at line 267 also gates the floatval locale check — must preserve that chain

## Tasks

### Task 1: Update index.php version guard

<read_first>
- index.php (lines 19-23)
</read_first>

<action>
In `index.php`, replace line 20-22:
```php
if ((float) substr(phpversion(), 0, 3) < 5.6) {
    /// comprobamos la versión de PHP
    die('FacturaScripts necesita PHP 5.6 o superior, y usted tiene PHP ' . phpversion());
}
```
With:
```php
if (version_compare(PHP_VERSION, '8.2', '<')) {
    die('FSFramework necesita PHP 8.2 o superior, y usted tiene PHP ' . PHP_VERSION);
}
```
</action>

<acceptance_criteria>
- `index.php` contains `version_compare(PHP_VERSION, '8.2', '<')`
- `index.php` no longer contains `5.6`
- `index.php` no longer contains `(float) substr(phpversion()`
- Message says "FSFramework necesita PHP 8.2 o superior"
</acceptance_criteria>
