# Plan 01-02: Remove Weak Random Fallback from install.php

## Objective

Remove the `str_shuffle()` based `random_string()` function and its fallback path in `random_secret_key()`. PHP 8.2+ always has `random_bytes()`, so the fallback is dead code that could be copied to other contexts and produce weak secrets.

## must_haves

- truths:
  - PHP 8.2+ always has `random_bytes()` — the fallback can never execute
  - `FS_SECRET_KEY` is used for CSRF tokens, signed URLs, and cookie signatures
  - The `random_string()` function is only called by `random_secret_key()` as a fallback
- risks:
  - If someone copies `random_string()` to another context on older PHP, it produces predictable output

## Tasks

### Task 1: Remove random_string() and simplify random_secret_key()

<read_first>
- install.php (lines 235-247)
</read_first>

<action>
In `install.php`, replace lines 235-247:

```php
function random_string($length = 20)
{
    return substr(str_shuffle("0123456789abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ"), 0, $length);
}

function random_secret_key($length = 64)
{
    if (function_exists('random_bytes')) {
        return bin2hex(random_bytes(intval($length / 2)));
    }

    return random_string($length);
}
```

With:

```php
function random_secret_key(int $length = 64): string
{
    return bin2hex(random_bytes(intdiv($length, 2)));
}
```

Remove the `random_string()` function entirely.
</action>
