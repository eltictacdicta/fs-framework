# Plan 01-03: Remove PHPMailer 5.x and Compat Bridge

## Objective

Remove the vendored PHPMailer 5.x code in `extras/phpmailer/` and the compatibility bridge in `extras/phpmailer_compat.php`. Update the one legacy caller (`empresa.php`) to use the namespaced PHPMailer 6.x class directly. Remove the bridge loading from `Kernel.php`.

## must_haves

- truths:
  - `composer.json` depends on `phpmailer/phpmailer:^6.0` — the 6.x class is always available
  - `src/Core/MailService.php` already uses the namespaced `PHPMailer\PHPMailer\PHPMailer` correctly
  - `plugins/business_data/model/empresa.php:459` uses `new \PHPMailer()` — relies on compat bridge
  - The compat bridge (`class_alias`) maps `PHPMailer` → `PHPMailer\PHPMailer\PHPMailer`
  - No other code in the project uses the legacy class names (`PHPMailer`, `SMTP`, `phpmailerException`)
- risks:
  - Third-party plugins might rely on the compat bridge — but those plugins should use the Composer namespace
  - `empresa.php` docblock says `@return \PHPMailer` — must update to `@return \PHPMailer\PHPMailer\PHPMailer`
