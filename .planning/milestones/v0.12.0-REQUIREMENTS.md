# Requirements: FSFramework — Security Audit & Hardening

**Defined:** 2026-05-23
**Milestone:** v0.12.0
**Core Value:** Fix real issues with minimal risk. Every change must be verifiable by the existing test suite and must not break plugins that depend on current behavior.

## v1 Requirements

### Audit — Baseline

- [x] **AUDIT-01**: Automated scan (9 FSFramework security categories) over core + 5 core plugins; output in `.planning/security/BASELINE-AUDIT.md`
- [x] **AUDIT-02**: Consolidate prior findings (02-REVIEW, CONCERNS) with open/fixed/deferred status
- [x] **AUDIT-03**: Classify each finding as CRITICAL / HIGH / MEDIUM / INFO with file:line reference

### CSRF — Mutation Protection

- [x] **CSRF-01**: `pre_private_core()` blocks invalid POST mutations (not only `$csrf_valid = false`)
- [x] **CSRF-02**: Remove double-validation that consumes token (e.g. `ventas_clientes`)
- [x] **CSRF-03**: `admin_home` and core controllers with state-changing POST verify `isCsrfValid()` before acting
- [x] **CSRF-04**: Regression tests for unified CSRF policy

### Injection & Input

- [x] **INJ-01**: Zero direct SQL concatenation with user input without `var2str`/prepared statements in scope
- [x] **INJ-02**: Replace direct `$_GET`/`$_POST`/`$_REQUEST` in scoped controllers with Request/filter helpers
- [x] **INJ-03**: Audit `|raw` in scoped Twig; justify or remove uses with untrusted data

### Headers & Production

- [x] **HDR-01**: CSP: plan to remove `unsafe-inline` (nonce-first); minimum: fix `connect-src` for CDNs
- [x] **HDR-02**: DebugBar restricted by IP or disabled outside local even when `FS_DEBUG=true`
- [x] **HDR-03**: Verify security headers in `src/Security/SecurityHeaders.php` with tests

### API — api_base Surface

- [x] **API-01**: Audit auth middleware, rate limit, and CORS in `plugins/api_base/` after core delegation
- [x] **API-02**: Verify `api.php` returns safe 404 when plugin inactive (no stack leak)
- [x] **API-03**: Review token/API key management in plugin admin models

### Authentication & Session

- [x] **AUTH-01**: Verify session regeneration post-login and cookie policy (httponly, secure, samesite)
- [x] **AUTH-02**: Document and validate legacy password flow in `legacy_support` (argon2id migration)
- [x] **AUTH-03**: Confirm `SafeRedirect` used correctly in post-action redirects

### Documentation

- [x] **DOC-01**: Publish `SECURITY.md` with threat model, phase mitigations, and REQ status
- [x] **DOC-02**: Run `/gsd-secure-phase` at closure of each remediation phase
