# Plan 01: Extract Plugin Management from admin_home

Extract `install_system_updater` + its 4 private helper methods into a `PluginInstaller` class, and extract plugin action routing from `exec_actions()` into a `PluginActionHandler` class. `admin_home` delegates instead of hosting the logic.

## must_haves

- `src/Core/PluginInstaller.php` — self-contained class with `installSystemUpdater()` + download/extract/move/verify helpers
- `controller/admin_home.php` — `install_system_updater()` is a one-line delegation to `PluginInstaller`
- `controller/admin_home.php` — `exec_actions()` plugin routing delegates to `PluginActionHandler` for enable/disable/delete/install/restore actions
- Public methods preserved: `admin_home->delete_plugin()`, `admin_home->enable_plugin()`, etc. still work (3-line delegations kept)
- Full test suite passes (381 tests, 0 failures, 0 errors, 14 skipped)

## Verification

```bash
ddev exec php vendor/bin/phpunit
```

Must exit with same counts as pre-change baseline.

## <threat_model>
- **Threats:** Low — code extraction, not new logic. The install_system_updater downloads from a hardcoded GitHub URL over HTTPS (existing behavior preserved). No new I/O, no credential handling added.
- **Data flow:** No change — the extracted classes call the same curl/file operations as before.
- **ASVS L1:** V5.1.1 (input validation) — existing input validation via filter_input() preserved in exec_actions().
</threat_model>
