# Plan 01: fs_mysql Decomposition

Decompose the 1577-line monolithic `fs_mysql` class into 3 focused classes: `TypeNormalizer` (pure functions), `SchemaInspector` (read-only introspection), and `SchemaComparator` (DDL generation). Extract in dependency order: pure → read-only → mutations. `fs_mysql` retains core DB operations and delegates schema work.

## must_haves

- `src/Database/TypeNormalizer.php` — pure static utility class with no DB dependency
- `src/Database/SchemaInspector.php` — read-only schema introspection (get_columns, get_constraints, get_indexes)
- `src/Database/SchemaComparator.php` — schema comparison and DDL generation (compare_columns, compare_constraints, generate_table)
- `base/fs_mysql.php` — delegates schema operations to extracted classes; retains connect/select/exec/transactions
- Full test suite passes (same counts as Phase 6: 383 tests, 0 failures, 0 errors)
- XML schema system continues working (model/table/*.xml → DB schema generation)

## Verification

```bash
ddev exec php vendor/bin/phpunit
```

Must exit with 0 failures, 0 errors. Base test suite (tests/Base/) must pass all tests unchanged.

## <threat_model>
- **Threats:** Medium — this is the deepest refactor of the phase, affecting DB schema operations that could break table creation/migration. Mitigations: (a) extract pure functions first (no side effects), (b) extract in 3 waves, verifying test suite after each, (c) fs_mysql keeps all public method signatures unchanged for backward compat.
- **Data flow:** No change. Schema data flows from XML files → fs_mysql → extracted classes. Connection/session state remains in fs_mysql.
- **ASVS L1:** V5.1.2 (SQL injection) — existing parameterized query infrastructure in fs_mysql unchanged. Extracted methods use the same quoting/escaping helpers.
</threat_model>
