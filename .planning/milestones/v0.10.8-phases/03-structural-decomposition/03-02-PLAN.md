# Plan 03-02: Decompose StealthMode into Focused Services

## Objective

Split `StealthMode.php` (1073 lines, 55 methods) into three focused services: `StealthAccessGate` (access logic), `HtmlSanitizer` (HTML sanitization), and `CssSanitizer` (CSS sanitization using sabberworm/php-css-parser). The original `StealthMode` class becomes a thin composition facade for backward compatibility.

## must_haves

- truths:
  - `StealthMode.php` is instantiated at `index.php:103` on every request
  - The class has 55 methods across 4 concerns: stealth access (lines 114-282), login integration (440-551), HTML sanitization (562-690), CSS sanitization (693-1002)
  - HTML/CSS sanitization uses `sabberworm/php-css-parser` with recursive tree traversal
  - Public API methods called from `index.php` and controllers must remain unchanged
  - Backward compatibility is critical â€” plugins may reference `StealthMode` directly
- risks:
  - Breaking the public API would break the stealth/admin panel
  - CSS sanitization is complex and entangled with the CSS parser library
  - The `StealthMode` class is instantiated on every request â€” decomposition must not add overhead
