# Plan 03-03: Extract Schema Ops from fs_mysql + Split admin_home

## Objective

Extract schema comparison and normalization logic from `fs_mysql.php` (1596 lines, 72 methods) into a dedicated `FsMysqlSchema` class. Split cache management functionality from `admin_home.php` (1053 lines) into a dedicated `admin_cache` controller.

## must_haves

- truths:
  - `fs_mysql.php` has 72 methods across concerns: connection management, query execution, escaping, schema comparison, constraint normalization, column type mapping
  - Schema operations (`compare_columns`, `compare_constraints`, `generate_table`, normalization methods) are ~15 methods that compose the schema diff/comparison engine
  - `admin_home.php` has 32 methods: dashboard rendering, plugin management UI, system info, cache clearing, update checking
  - Cache clearing logic in `admin_home` handles legacy cache, Twig cache, file cache, and memcache
  - Both `fs_mysql` and `admin_home` are used from many places — public API must not change
- risks:
  - `fs_mysql` is the core database driver — any change risks breaking all DB operations
  - Schema comparison is deeply coupled to the driver's internal data structures
  - `admin_home` is referenced by menu items and controller routing
