# Plan 02-02: Add Test Coverage for business_data and catalogo_core Plugins

## Objective

Add automated unit tests for business-critical models in `business_data` (empresa, ejercicio, serie, divisa) and `catalogo_core` (familia, fabricante). `articulo` already has a test. These tests verify model instantiation, property hydration, and basic validation — no database required.

## must_haves

- truths:
  - Tests use PHPUnit 11, run via `ddev exec php vendor/bin/phpunit`
  - Tests must NOT require database — use anonymous constructor or mock
  - Models extend `fs_model` which requires `$table_name` in constructor
  - `fs_model` constructor calls `$this->db = Container::db()` — need to skip DB
  - Proxy models (pais, divisa in business_data) require the real class from catalogo_core
  - Existing test at `plugins/catalogo_core/tests/ArticuloModelEncodingTest.php` is the reference pattern
- risks:
  - `empresa` and `articulo` are complex models (558 and 1391 lines) — test only instantiation + basic methods, not full business logic
  - `ejercicio` links to `empresa` via `codejercicio` — keep tests independent
  - Proxy models need `require_once` for the real class file

Create `plugins/catalogo_core/tests/FamiliaModelTest.php` following the pattern from `ArticuloModelEncodingTest.php`:

```php
<?php
/**
 * [standard FSFramework license header]
 */

namespace Tests\CatalogoCore;

use PHPUnit\Framework\TestCase;

class FamiliaModelTest extends TestCase
{
    protected function setUp(): void
    {
        global $plugins;
        $plugins = [];

        require_once FS_FOLDER . '/base/fs_model.php';
        require_once FS_FOLDER . '/plugins/catalogo_core/model/core/familia.php';
    }

    public function testConstructorHydratesFromArray(): void
    {
        $data = [
            'codfamilia' => 'FAM001',
            'descripcion' => 'Familia de prueba',
            'madre' => null,
        ];
        $model = $this->createModel($data);
        $this->assertSame('FAM001', $model->codfamilia);
        $this->assertSame('Familia de prueba', $model->descripcion);
        $this->assertNull($model->madre);
    }

    public function testConstructorWithNumericIdSkipsLoading(): void
    {
        $model = $this->createModel(1);
        $this->assertIsObject($model);
    }

    public function testConstructorWithNull(): void
    {
        $model = $this->createModel(false);
        $this->assertIsObject($model);
    }

    private function createModel($data = false): object
    {
        return new class($data) extends \familia {
            public function __construct($d = false)
            {
                parent::__construct($d);
            }
        };
    }
}
```

Adapt as needed based on the actual `familia` class structure (properties, constructor behavior).
</action>
