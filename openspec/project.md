## Conventions (from `AGENTS.md`)

- Mandatory environment: **DDEV** for all PHP / Composer / PHPUnit commands. `ddev exec php ...` not raw `php`.
- Assume `ddev start` is running when tests or DB-backed services are needed.
- Stack baseline: PHP 8.2+ (prefer 8.3), Symfony 7.4, Twig 3, PHPUnit 11.
- Architecture split: legacy `base/`+`controller/`+`model/` vs modern PSR-4 `src/`. Prefer modern Symfony services; keep legacy compatibility in thin bridges/wrappers.
- Prefer Twig for new views; Symfony services and the container for modern code.
- Safe SQL (`var2str`, prepared statements), escaped output (`{{ }}`), CSRF in mutating flows, secure password handling, safe file handling.
- Add or update tests for business logic changes, especially reusable logic in `src/` and plugins.
- Plugin conventions: `fsframework.ini` with `min_version` and `require` dependencies; plugin-specific test prefix in translation keys; `declare(strict_types=1)` in modern PHP files.
- Never commit `config.php`, `package-lock.json`, or `node_modules/`.
